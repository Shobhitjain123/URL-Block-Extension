# Focus Lock — Demo Video Shot List

Optional 30–60 second screen recording for the Chrome Web Store listing. Upload to YouTube (unlisted is fine) and paste the URL in the store form.

## Setup

1. Load the extension unpacked from this project root in `chrome://extensions`.
2. Use a harmless test domain you control, or a domain you are okay blocking temporarily (for example, a news site).
3. Use QuickTime (macOS), OBS, or Chrome’s built-in screen recorder.
4. Target resolution: **1280×800** or **1920×1080**.

## Shot list

| # | Duration | Action | Narration (optional) |
| --- | --- | --- | --- |
| 1 | 5s | Show Chrome toolbar; click the Focus Lock icon | “Focus Lock helps you block distracting sites for a fixed time.” |
| 2 | 8s | Type a domain (e.g. `example.com`), select **2h**, add an optional reason, click **Lock it** | “Enter any URL — the whole domain is blocked, including subdomains and paths.” |
| 3 | 5s | Show the **Active blocks** list with live countdown | “There’s no unlock button. The timer is the only way out.” |
| 4 | 10s | Open a new tab; navigate to the blocked domain (or a path like `/browse`) | “When you visit the site, you see a calm blocked page instead.” |
| 5 | 8s | Highlight the countdown and optional reason on `blocked.html` | “Your reason stays visible until the block expires.” |
| 6 | 4s | Return to the popup; show the same block still counting down | “Blocks survive extension reloads and browser restarts.” |

## Export

- Trim dead air at start/end.
- Export as MP4.
- Upload to YouTube → set visibility to **Unlisted**.
- Copy the video URL into the Chrome Web Store listing under **Promotional video**.

## What not to include

- Personal browsing history or sensitive tabs.
- Real passwords or account details in any visible field.
